The Odoo Email Connector add-on respects your privacy. This add-on does not collect, track, or transmit personal data to any third party. Data Access The add-on accesses email content, headers, and attachments when explicitly triggered by the user via the Thunderbird context menu (import, verify, and sync actions). Additionally, when viewing an email, the add-on automatically reads the message identifier (Message-ID header) to look up cached Odoo status (see below). Accessed data is used solely for the purpose of importing or verifying emails against the user's own Odoo server.

When viewing an email, the add-on automatically reads the Message-ID header to look up the Odoo status from the local cache and display the status bar. No data is transmitted during this process; only locally cached information is used. Data Transmission Email data and Message-IDs are transmitted directly from Thunderbird to the Odoo server configured by the user. No data is sent to the add-on developer or to any external service other than the configured Odoo instance. Transmission occurs during explicit user actions (import, verify) and during the user-initiated Sync from Odoo feature, which fetches message status from the Odoo server in the background. Local Storage The add-on stores configuration data locally in Thunderbird, such as:
  • Odoo server URL
  • Odoo database name (if required)
  • Odoo API key
  • User-selected configuration options
This information is stored locally using Thunderbird's extension storage and is never shared with third parties. Third-Party Services The add-on does not use analytics, tracking, advertising, or third-party telemetry services. User Control Import and verify actions require explicit user interaction. The Sync from Odoo action is triggered by the user and then runs in the background to fetch message status from the Odoo server. When viewing an email, the Message-ID header is read automatically to display cached Odoo status (no data is transmitted during this automatic lookup). Removing the add-on will remove all locally stored configuration data.
